# Security Policy

## Supported Versions

This project is an event-based website and is not versioned for long-term releases.  
The latest version in the `main` branch is considered the active and supported version.

---

## Reporting a Vulnerability

If you discover a security vulnerability in this project, please report it responsibly.

### How to report

- Email: security@shoug-tech.com 
- Or open a **private report** (do NOT open a public issue for sensitive vulnerabilities)

### What to include

Please provide as much detail as possible:

- Description of the vulnerability  
- Steps to reproduce the issue  
- Potential impact  
- Screenshots or proof-of-concept (if applicable)

---

## Guidelines

- Do not publicly disclose the vulnerability until it has been reviewed and addressed  
- Avoid exploiting the vulnerability beyond what is necessary to demonstrate it  
- Be respectful and responsible in your reporting  

---

## Scope

This repository contains a static website built with MkDocs.  
Potential areas of concern may include:

- Custom JavaScript (`docs/overrides/main.js`)
- HTML overrides (`docs/overrides/main.html`)
- External links or embedded resources

---

## Response

We will review reported issues as soon as possible.  
Since this is an academic project, response times may vary.

---

## Notes

This project does not currently include:

- user authentication systems  
- databases  
- backend services  

As such, security risks are limited to front-end behavior and third-party integrations.
